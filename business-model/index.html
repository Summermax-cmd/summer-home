商业模式演进 · 房源投放系统

贝壳商业模式 Before → After

在「挂房源—带看—议价—成交」的存量链路之上，新增一套房源投放系统：房价与佣金率不变，投放对应的是增值服务，为平台打开新的收益机会点。

BEFORE

传统居住交易链路

货架浏览 · 经纪人撮合 · 线下成交
🙋买房用户 🧑‍💼经纪人 🏢平台 🏠卖房用户
卖方侧 卖房用户挂房源→ 经纪人维护房源→ 线下带看→ 议价→ 成交
买方侧 平台货架浏览房源→ 咨询联系经纪人→ 线下带看→ 议价→ 成交
🏢 平台从成交中获得  x%  分成（基于 ACN 协作网络）
房子价格 = y 佣金点位 = z% ACN 经纪人协作网络
+
新增能力：房源投放系统
在原链路之上叠加「投前 → 投中 → 投后」闭环，不改变价格与佣金，只新增增值服务
AFTER

叠加房源投放系统

出价驱动 · 放大核访 · 投后反馈优化
🙋买房用户 / 经纪人 🏠卖房用户 / 经纪人 📡房源投放系统新增 🏢平台
投前
  • 卖方用户 / 经纪人出价
  • 价不止是货币，付出的时间与精力成本也计入
  • 平台据出价匹配准客 / 潜客
投中
  • 在分发场景放大该房源的核访
  • 配备专属触达素材 / 落地页 / 服务
  • 买方发现投放房源的概率更高
投后
  • 获得投后反馈（核访、留资、带看…）
  • 按效果分两路（见下）
  • 为投放房源带来更多成交机会

① 有效果

带来更多客户 → 提升成交机会，可继续 / 加投

② 无效果

系统给出优化建议 → 调整出价 / 换房源 / 调周期

↻ 投后反馈回流到下一轮出价，形成投前 → 投中 → 投后持续闭环
🙋 买方侧：投放让买房用户 / 经纪人更容易发现对的房源，为投放房源带来更多成交机会；平台仍从成交中获得 x% 分成。
关键

不变的 与 新增的

投放 = 增值服务，是平台新的收益机会点
保持不变

房价 与 佣金率 不动

投放系统不改变房子的成交价格、也不改变佣金点位，原有交易关系与 ACN 协作完全保留。

房子价格y🔒 不变
佣金率z%🔒 不变
新增价值

投放 = 增值服务 → 平台新收益

在不动价格 / 佣金的前提下，投放是额外的增值服务：卖方用出价换「确定性的曝光与核访放大」。

  • 对卖方：花钱 / 花精力换更高成交确定性
  • 对买方：更快找到对的房
  • 对平台：在 x% 分成之外，开辟增值服务的新收益机会点
