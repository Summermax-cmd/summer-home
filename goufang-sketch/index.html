01

项目在决策 OS 中的定位

消费者决策 OS 有两个核心模型：买家侧的"引力式决策"和卖家侧的"到站式决策"。购房计划书服务的是买家引力式决策——帮买家构建需求画像、完成匹配评估和取舍判断，最终生产出"准客"。准客从卖家"曝光"环节起持续汇入，贯穿曝光、带看、出价、斡旋成交全链路。

买家引力式决策

核心逻辑：买家带着需求画像（预算 / 家庭 / 通勤 / 学区 / 改善 / 审美），被匹配的房源产生引力或排斥。不是线性流程，而是多重力同时作用——被吸引、被排除，反复权衡后做出最优选择。

购房计划书的价值：它是卖家引力模型的产品化落地——帮买家分析自身需求、形成清晰的描述，通过服务者协同完成修改、决策、确认，从流量成为准客。准客的需求汇入卖家链路，同时提升买卖两侧的效率。

03

关键设计

关键设计不是堆叠功能，而是把用户原本散落在线下的"探需—对比—判断"动作，放进一条线上可执行、可沉淀的规则里，让泛流量层层收敛成"准客"。

过去

用户浏览 → 翻找货架 → 产生咨询 → 随机派单 → 盲目带看 → 需求未沉淀，平均 1 个成交用户要联系 16 个经纪人。

现在

用户浏览 → 互动探需 → 生成《购房计划书》→ 自主确认 → 精准派单 → 需求进一步沉淀。

机制设计 · 背后的原因

计划书做中间件"委托 / 带看"只是动作，计划书衡量的是需求收敛与决策深度，才是真正的准客里程碑。
准客严格口径口径为"被分类 + 自确认"，告别一视同仁的概率运营，把资源集中到真实意向用户。
AI 管家先承接不急于推线下，AI 在前端完成最费时的需求教育和预期管理。
博弈库透传透传"大家还在看""意向明细"等真实行为信号，缓解情报荒。
能人精准派单重建带看—成交因果，缩短转化周期、提升二次带看率。

决策阶段 · 含义

泛浏览隐性需求、未被分类，资源容易浪费在低意向用户上。
互动探需7×24h AI 管家承接，通过产品互动及在线问答让需求开始显性化。
购房计划书结构化需求 + 决策辅助，完成需求收敛，达到准客门槛。
自主确认服务者协同，用户对需求完成修改、决策、确认，正式成为准客。
派单能人带着计划书进入线下，重建"带看—成交"的因果关系。
