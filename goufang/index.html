BC 协同做大准客规模

以购房计划书为枢纽，让平台线上能力与经纪人线下能力形成协同，把泛流量转化为可被清晰描述的需求

项目在决策 OS 中的定位

消费者决策 OS 有两个核心模型：买家侧的"引力式决策"和卖家侧的"到站式决策"。购房计划书服务的是买家引力式决策——帮买家构建需求画像、完成匹配评估和取舍判断，最终生产出"准客"。准客从卖家到站式决策的"曝光"环节起就持续汇入，贯穿曝光、带看、出价、斡旋成交全链路。

买家引力式决策

核心逻辑：买家带着需求画像（预算/家庭/通勤/学区/改善/审美），被匹配的房源产生引力或排斥。这不是线性流程，而是多重力同时作用——买家在决策视野内被吸引、被排除，反复权衡后做出最优选择。

准客从"曝光"开始就汇入卖家链路：

准客带着明确需求和决策意向，从"曝光"站点起就为卖家提供精准客户信息——曝光、带看、出价、斡旋成交每一站都需要准客信息来提升匹配精度、去化效率和成交确定性。

购房计划书的价值：它是卖家引力模型的产品化落地——帮买家分析自身需求、形成清晰的描述，通过服务者协同完成修改、决策、确认，从流量成为准客。准客的需求汇入卖家链路，同时提升买卖两侧的效率。

关键设计

关键设计不是堆叠功能，而是把用户原本散落在线下的"探需—对比—判断"动作，放进一条线上可执行、可沉淀的规则里，让泛流量层层收敛成"准客"。

过去用户浏览 → 翻找货架 → 产生咨询 → 随机派单 → 盲目带看 → 需求未沉淀，平均 1 个成交用户要联系 16 个经纪人。
现在用户浏览 → 互动探需 → 生成《购房计划书》→ 自主确认 → 精准派单 → 需求进一步沉淀。
